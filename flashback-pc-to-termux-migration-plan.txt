### 4.2 Screenshot Worker (`screenshot_worker.py`)
**Replace PC screenshot methods with Termux primitives:**

```python
from termux_screenshot_workers_primitives import (
    check_su_available,
    get_current_user,
    take_screenshot
)

class TermuxScreenshotWorker:
    def __init__(self, output_dir):
        self.output_dir = output_dir
        self._check_prerequisites()

    def _check_prerequisites(self):
        if not check_su_available():
            raise RuntimeError("Root access (su) required for Termux screenshot worker")
        # Ensure output directory exists and is writable

    def capture(self, filename=None):
        """
        Capture screenshot and save to output_dir.

        Args:
            filename: Optional filename, otherwise auto-generate with timestamp

        Returns:
            Path to saved screenshot or None on failure
        """
        if filename is None:
            from datetime import datetime
            filename = f"screenshot_{datetime.now().strftime('%Y%m%d_%H%M%S')}.png"

        output_path = os.path.join(self.output_dir, filename)
        success, message = take_screenshot(output_path)

        if success:
            return output_path
        else:
            logger.error(f"Screenshot failed: {message}")
            return None
```

### 4.3 Window Title Worker (`window_title_worker.py`)
**Replace PC window title detection with Android app detection:**

```python
from termux_screenshot_workers_primitives import (
    check_su_available,
    get_active_app_info,
    get_app_name_by_id
)

class TermuxWindowTitleWorker:
    """
    Detect active Android app for screenshot context.

    Fallback hierarchy:
    1. Try to get app name from active activity
    2. If app name is None, use APK ID (package name)
    3. If all detection fails, return "Unknown"
    """

    def __init__(self):
        self._check_prerequisites()
        self._cache = {}  # Cache app_id -> app_name lookups

    def _check_prerequisites(self):
        if not check_su_available():
            raise RuntimeError("Root access (su) required for window title worker")

    def get_current_context(self):
        """
        Get current active app context.

        Returns:
            Dict with keys:
                - app_name: Human-readable app name or None
                - app_id: Package name (e.g., "com.android.chrome")
                - display_title: Final title to use (app_name > app_id > "Unknown")
        """
        app_info = get_active_app_info()

        app_id = app_info.get("package", "")
        activity = app_info.get("activity", "")

        if not app_id:
            return {
                "app_name": None,
                "app_id": None,
                "display_title": "Unknown",
                "activity": None
            }

        # Try to get human-readable app name
        app_name = self._get_app_name(app_id)

        # Determine display title (fallback hierarchy)
        if app_name:
            display_title = app_name
        else:
            display_title = app_id  # Use package name as fallback

        return {
            "app_name": app_name,
            "app_id": app_id,
            "display_title": display_title,
            "activity": activity
        }

    def _get_app_name(self, app_id):
        """
        Get human-readable app name from package ID.

        Uses cache to avoid repeated lookups.

        Args:
            app_id: Package name (e.g., "com.android.chrome")

        Returns:
            App name string or None
        """
        if app_id in self._cache:
            return self._cache[app_id]

        app_name = get_app_name_by_id(app_id)
        self._cache[app_id] = app_name
        return app_name
```

### 4.4 Integration Points
- Replace imports: `flashback_screenshots` -> `flashback_termux`
- Replace config paths: `~/.config/flashback-screenshots` -> `~/.config/flashback-termux`
- Replace default screenshot directory: PC Pictures folder -> Termux home or `/sdcard/Pictures/flashback`
- Add dependency on `termux-screenshot-workers-primitives`

### 4.5 Error Handling Strategy
- If `check_su_available()` fails on startup: Show clear error message about requiring root
- If screenshot fails: Log error, continue (don't crash)
- If app detection fails: Use "Unknown" as context
- If app name lookup fails: Use app_id as display_title
